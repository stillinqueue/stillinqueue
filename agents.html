Drag-and-Drop Workflows

SQ-AI takes inspiration from n8n and similar tools, but is designed around agents instead of raw APIs. Later, this canvas becomes where you link agents, data sources, and notifications.

Node Palette (Concept)

The workflow starts with triggers (time, data, or events), flows through agents that add logic, and ends in actions like notifications, webhooks, or updates to your tools.

Trigger Schedule · Webhook · Event
Agent Stock · Travel · Health · etc.
Action Email · Chat · API Call
Logic Filters · Branches · Loops

Under the hood this will run on a Python stack (e.g. FastAPI / Django) so you can connect databases, dashboards, and external APIs directly.

Signals & Notifications

Alerts that Respect Your Attention

Instead of constant noise, SQ-AI focuses on trend-based, rule-based, and exception-based alerts so you only hear from agents when it actually matters.

Trend-Based Alerts

Configure agents to react to patterns instead of single events: moving averages in markets, multi-day price drifts in flights, or week-over-week changes in your habits.

Example · Stock · Flights · Habits

Rule-Based & Thresholds

Simple “if this then that” rules with AI helping you design good thresholds. For example: “Alert me if monthly expenses cross X% of income for two weeks in a row.”

Example · Money · Energy · Time

Streams & Digest Views

Choose between real-time streams (chat-style) or calm digests delivered daily or weekly. Later, these will link into email, chat apps, and a dedicated SQ-AI console.

Example · Daily · Weekly · Custom

About · SQ-AI

The Vision Behind SQ-AI

SQ-AI is the first fold of Still In Queue — a long-term vision to harness AI and unify technology, learning, sustainability, and coffee into connected digital experiences. SQ-AI focuses on workflows and agents: practical building blocks that can watch trends, react to signals, and help with daily decisions.

This page is the design layer for the future console. In upcoming phases, it will connect to a Python backend (FastAPI / Django), data engineering pipelines, real-time notifications, and a secure login system — turning these concepts into live tools that people can rely on.

Roadmap idea:

Phase 1: Front-end and information architecture.
Phase 2: User accounts, basic agents, manual triggers.
Phase 3: Full workflow engine with scheduling and integrations.
Phase 4: Marketplace of community-built agents and templates.
